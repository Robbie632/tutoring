Client-Server Pattern: Browser Downloads HTML, CSS, and JavaScript

This page uses Mermaid.js to visualize how a web browser (client) interacts with a web server to download HTML, CSS, and JavaScript files. HTTP status codes are shown to indicate the result of each request.

Diagram showing how a browser loads html, javascript and html

sequenceDiagram actor Browser participant Server Browser->>Server: GET /index.html Server-->>Browser: 200 OK (HTML) Browser->>Server: GET /styles.css Server-->>Browser: 200 OK (CSS) Browser->>Server: GET /app.js Server-->>Browser: 200 OK (JavaScript) Browser->>Server: GET /missing.js Server-->>Browser: 404 Not Found

Status Codes Explained

  • 200 OK: The request was successful and the resource (HTML, CSS, JS) is returned.
  • 404 Not Found: The requested resource does not exist on the server.
  • 500 Internal Server Error: The server encountered an error while processing the request.
